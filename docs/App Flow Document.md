## **App Flow Document** 

## **Portfolio Website – Shib Chandan Mistry** 

## **1. Objective** 

Define how users (recruiters, engineers) navigate through the portfolio to: 

- Quickly understand your profile 

- Explore projects 

- Dive into technical depth 

- Contact you 

## **2. Primary User Flow (Recruiter Journey)** 

## **Step-by-Step Flow** 

1. Landing on Homepage 

2. Scanning Hero Section (3–5 seconds) 

3. Scrolling to Projects 

4. Clicking 1–2 projects 

5. Viewing details (modal/page) 

6. Checking skills 

7. Visiting GitHub or contacting 

## **3. High-Level Navigation Flow** 

Landing Page 

↓ 

Hero Section 

↓ 

Projects Section 

↓ 

Project Detail (Modal/Page) 

↓ 

Skills Section 

↓ 

Contact Section 

## **4. Detailed Screen Flow** 

## **4.1 Landing Page** 

## **Entry Point:** 

- Direct URL (recruiter link) 

- GitHub profile 

- Resume link 

## **Actions:** 

- Scroll down 

- Click "View Projects" 

- Click Resume 

## **4.2 Hero Section Flow** 

## **User Actions:** 

- Read name + role 

- Evaluate credibility 

## **Exit Paths:** 

- Scroll down → Projects 

- Click CTA → Projects 

## **4.3 Projects Section Flow (Core Interaction)** 

## **Grid View:** 

- User scans project cards quickly 

## **Decision Point:** 

- “Is this interesting?” 

## **Actions:** 

- Click project → Open detail view 

- Click GitHub → External navigation 

- Click Demo → Video / demo 

## **4.4 Project Detail Flow** 

## **Trigger:** 

Click on project card 

## **Opens:** 

- Modal OR separate page 

## **User Journey Inside:** 

1. Read simplified description 

2. Understand problem 

3. Skim features 

4. Check tech stack 

5. Optional deep dive 

## **Exit:** 

- Close modal 

- Go to another project 

## **4.5 Skills Section Flow** 

## **Purpose:** 

## Validate your technical breadth 

## **User Behavior:** 

- Quick scan (5–10 seconds) 

## **4.6 Contact Section Flow** 

## **Actions:** 

- Click GitHub 

- Click LinkedIn 

- Copy email

## **6. Alternate Flows** 

## **6.1 Fast Recruiter Flow** 

Hero → Projects → GitHub → Exit 

## **6.2 Technical Interviewer Flow** 

Hero → Projects → Deep Dive → GitHub → Multiple projects 

## **6.3 Mobile Flow** 

- Scroll-based navigation 

- Single-column project view 

- Tap → modal 

## **7. UX Optimization Points** 

## **Critical Enhancements** 

- Projects visible within first scroll 

- Clear CTA buttons 

- No unnecessary navigation layers 

- Instant modal loading 

## **8. State Flow (Technical)** 

## **States:** 

- Default (homepage) 

- Project modal open 

- External redirect (GitHub/demo) 

## **9. Navigation Rules** 

- Single-page scroll-based navigation 

- Anchor links (smooth scroll) 

- No complex routing initially 

## **10. Conversion Points** 

Where user takes action: 

- View Projects (primary) 

- Open Project Modal (engagement) 

- Click GitHub (validation) 

- Contact (final conversion) 

## **11. Performance Flow Considerations** 

- Lazy load project images 

- Preload modal content 

- Avoid blocking animations 

## **12. Final Insight** 

The app flow must: 

- Minimize thinking 

- Maximize clarity 

- Guide user naturally to projects 

The entire experience should feel like: “Scan → Interest → Proof → Contact”
